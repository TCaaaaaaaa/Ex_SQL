Algorithm 1: SQL 差异分析与纠错建议生成 (SQL Diagnosis)
Input: user_sql, std_sql
Output: suggestions (List of String)

1. M_user = ParseToMatterElement(user_sql)  // 解析用户SQL为物元
2. M_std = ParseToMatterElement(std_sql)    // 解析标准SQL为物元
3. diffs = []
4. suggestions = []

// 检查 SELECT 子句
5. For each column c_std in M_std.select_clause:
6.     If c_std not in M_user.select_clause:
7.         diffs.append(MISSING_COLUMN, c_std)
8.         suggestions.append("建议在 SELECT 子句中增加列: " + c_std)

// 检查 FROM 子句
9. For each table t_user in M_user.from_clause:
10.    If t_user not in M_std.from_clause:
11.        diffs.append(REDUNDANT_TABLE, t_user)
12.        suggestions.append("建议从 FROM 子句中移除多余的表: " + t_user)

// 检查 WHERE 子句
13. For each condition cond_std in M_std.where_clause:
14.    If cond_std not in M_user.where_clause:
15.        If FindSimilar(cond_std, M_user.where_clause):
16.            diffs.append(WRONG_CONDITION, cond_std)
17.            suggestions.append("建议修正 WHERE 子句中的条件: " + cond_std)
18.        Else:
19.            diffs.append(MISSING_CONDITION, cond_std)
20.            suggestions.append("建议在 WHERE 子句中增加筛选条件: " + cond_std)

21. Return suggestions

Algorithm 2: 基于关联函数的 K 值更新 (Update K-Value)
Input: current_k, is_correct (Boolean), time_cost (Float)
Output: new_k

1. Constants:
   W1 = 0.7   // 正确性权重
   W2 = 0.3   // 时间效率权重
   DECAY = 0.9 // 记忆衰减因子
   BIAS = 0.5 // 基础偏差

2. score = is_correct ? 1.0 : -1.0
3. time_score = (STANDARD_TIME - time_cost) / STANDARD_TIME
4. k_instance = (W1 * score) + (W2 * time_score) - BIAS

5. new_k = (current_k * DECAY) + k_instance
6. Return new_k
