# 实验设计与系统评估 (Experimental Design & Evaluation)

本文档定义了验证本系统有效性的实验方案，包括评估指标、实验流程及预期结果。

## 1. 实验目标

验证基于可拓学与大模型融合的 ExtenicSQL 系统在 **诊断准确性** 和 **教学辅助有效性** 两个维度上优于传统方法。

## 2. 评估指标体系 (Evaluation Metrics)

### 2.1 诊断准确率 (Diagnosis Accuracy)

针对 500 个包含典型错误的 SQL 样本（覆盖语法错、逻辑错、性能错）进行测试。

* **语义错误召回率 (Semantic Error Recall)**：

  $$
  R_{sem} = \frac{\text{系统检出的逻辑错误数}}{\text{样本中实际包含的逻辑错误总数}} \times 100\%
  $$

  * *Baseline*: 传统编译器 (MySQL/SQLite) 的逻辑错误召回率为 0% (仅能报 Syntax Error)。
  * *Target*: 本系统目标 > 85%。
* **变换建议有效性 (Suggestion Validity)**：

  $$
  V_{sugg} = \frac{\text{采纳建议后通过测试的样本数}}{\text{采纳建议的总样本数}} \times 100\%
  $$

### 2.2 学习路径转化率 (Learning Conversion Rate)

基于关联函数 $K(x)$ 的动态变化，评估系统对学生认知状态的干预效果。

* **正域转化率 (Positive Conversion Rate)**：
  $$
  R_{conv} = \frac{N(\text{state } -1 \to 1)}{N(\text{total attempts in Extension Domain})}
  $$

  * 衡量系统能否有效地将处于“可拓域”（临界状态）的学生推向“稳定正域”。

## 3. 实验方案 (Experimental Setup)

### 3.1 实验一：诊断能力对比实验

* **数据集**: Spider 数据集 (部分) + 教师人工构造的错题集 (ExtenicSQL-Error-Set)。
* **对照组**: 标准 SQLite 报错信息。
* **实验组**: ExtenicSQL 智能诊断反馈。
* **流程**:
  1. 输入错误 SQL。
  2. 记录两组系统的输出。
  3. 人工评估输出的准确性和可理解性 (1-5分打分)。

### 3.2 实验二：教学效果 A/B 测试

* **对象**: 100 名数据库课程初学者。
* **分组**:
  * **Group A (Control)**: 使用传统 OJ 平台，仅提示 Pass/Fail。
  * **Group B (Experiment)**: 使用 ExtenicSQL 平台，开启个性化推荐与可视化看板。
* **周期**: 2 周强化训练。
* **假设检验**:
  * **H1**: Group B 的平均解题尝试次数 (Average Attempts per Problem) 显著低于 Group A。
  * **H2**: Group B 在期末测试中复杂查询 (Complex Query) 的得分显著高于 Group A。

## 4. 可视化分析方法 (Visualization Analysis)

本系统内置的“数据看板”功能直接为实验分析提供了素材。

1. **个体微观分析 (Micro-Analysis)**

   * 使用 **动态知识图谱 (Force-Directed Graph)** 展示单个学生知识结构的演化。
   * *观察点*：观察连通子图的颜色变化，验证“前置知识点掌握后，后继知识点习得速度加快”的假设。
2. **群体宏观分析 (Macro-Analysis)**

   * 使用 **K值演化折线图** 对比两组学生的平均学习曲线。
   * *预期*：实验组的曲线斜率（学习速率）应大于对照组，且震荡幅度（试错成本）更小。
