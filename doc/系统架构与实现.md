# 系统架构与实现细节 (System Implementation Details)

## 1. 系统总体架构

采用前后端分离的 B/S 架构，后端集成大模型接口与数据库服务。

*   **前端 (Frontend)**: Vue 3 + TypeScript + Vite + Ant Design Vue
    *   **代码编辑器**: Monaco Editor (支持 SQL 语法高亮与智能提示)
    *   **可视化引擎**: Apache ECharts 5 (力导向图、折线图)
    *   **状态管理**: Pinia (持久化存储用户 K 值状态)
*   **后端 (Backend)**: Python FastAPI
    *   **SQL 执行沙箱**: Python `sqlite3` (内存模式 :memory:)
    *   **数据持久化**: SQLite + SQLAlchemy (异步模式)
    *   **智能接口**: OpenAI API / Local LLM Integration
*   **部署**: Docker 容器化部署

## 2. 关键模块实现

### 2.1 动态知识图谱 (Dynamic Knowledge Graph)
位于前端 `front/src/components/KnowledgeGraphChart.vue`。

*   **布局算法**: 采用 Force-Directed Layout (力导向布局)，模拟节点间的斥力与边的引力，实现拓扑结构的自动舒展。
*   **状态映射**:
    *   **节点颜色**: 绑定 `knowledgeStore` 中的 `state` 枚举 (STABLE_POSITIVE $\to$ 绿色, NEGATIVE $\to$ 红色)。
    *   **连线逻辑**: 遍历 `KnowledgeDependency` 字典，自动生成有向边。
*   **交互设计**: 支持拖拽重排、滚轮缩放、鼠标悬停显示 K 值详情。

### 2.2 SQL 执行与沙箱隔离 (Sandboxed Execution)
位于后端 `back/app/core/executor.py`。

为了防止恶意 SQL (如 `DROP TABLE users`) 破坏系统，采用 **内存数据库隔离** 机制：
1.  用户请求到达时，调用 `sqlite3.connect(":memory:")` 创建临时的纯内存数据库。
2.  注入题目预设数据 (`init_sql`)。
3.  执行用户 SQL，捕获结果或异常。
4.  比对逻辑：提取结果集的 `columns` 和 `values` 与标准答案进行值比对，而非字符串比对。
5.  请求结束，内存数据库自动销毁。

### 2.3 用户鉴权与状态同步
位于 `back/app/api/users.py` 与前端 `userStore.ts`。

*   **鉴权**: 简单的基于用户名/密码的认证机制 (Session/Token)。
*   **数据同步**: 
    *   前端 `localStorage` 实时缓存，保证刷新不丢失。
    *   关键动作 (通关/失败) 触发 `API.updateUserKnowledge`，将 JSON 格式的知识图谱全量同步至后端 SQLite 数据库。
