# 实验设计与系统评估指标体系

## 1. 实验目标
验证基于可拓学的 SQL 诊断系统在“错误定位准确率”和“教学辅助有效性”上优于传统方法。

## 2. 评估指标定义

### 2.1 诊断准确率 (Diagnosis Accuracy)
选取 500 个典型的学生错误 SQL 样本（包含语法错、逻辑错、性能错）。
*   **基线 (Baseline)**：传统编译器报错 (Syntax Error / Runtime Error)。
*   **本系统**：可拓学差异分析结果。
*   **指标**：
    *   **语义错误召回率 (Semantic Error Recall)**：系统能否发现诸如“范围错误”、“条件缺失”等编译器无法捕获的错误。
    *   **变换建议有效性 (Suggestion Validity)**：依据系统建议修改后，代码通过率。

### 2.2 学习路径转化率 (Learning Conversion Rate)
基于关联函数 $K(x)$ 的动态变化进行评估。
*   **定义**：在 $T$ 时间窗口内，学生知识点状态从“可拓域/负域”转化为“正域”的比例。
    $$ R_{conv} = \frac{N(\text{state } -1 \to 1)}{N(\text{total attempts})} $$

## 3. 验证实验设计 (A/B Testing)

### 3.1 实验设置
将 100 名学生随机分为两组，进行为期 2 周的 SQL 强化训练。
*   **对照组 (Control Group)**：使用标准 SQL 练习平台，仅提示 "Wrong Answer" 或标准 SQL 报错。
*   **实验组 (Experimental Group)**：使用本系统，提供基于可拓变换的“引导式反馈”。

### 3.2 假设检验
*   **H1**：实验组完成每道题的平均尝试次数 (Average Attempts) 显著少于对照组。
*   **H2**：实验组在期末测试中的复杂查询（Complex Query）得分显著高于对照组。

## 4. 可视化展示计划
在论文中展示以下图表：
1.  **知识状态演化图**：单个学生 $K(x)$ 值随做题数量的变化曲线（体现从震荡到稳定的过程）。
2.  **错误类型分布雷达图**：展示系统捕获的“缺少要素”、“要素多余”、“结构错误”等不同可拓变换类型的分布比例。
